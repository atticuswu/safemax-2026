# 2026 全域 SAFEMAX 動態提領與質押戰略指南

> **回測程式：** https://safemax.atticus.tw
> **最後更新：** 2026-04-10

---

## 一、核心底層邏輯：4.7% SAFEMAX

根據 Bill Bengen 2025 年的研究，**4.7%** 是通過百年歷史壓力測試（含大蕭條、惡性通膨）的「全域安全最大提領率」。

- **基準點：** 4.7% 是歷史上的「地板」，對應最差的退休時點（如 1968 年）。
- **兩大變數：** 退休開場時的席勒 CAPE 值與前五年通膨率，決定個人 SAFEMAX 能否向平均值 7.1% 靠攏。

---

## 二、動態提領框架：每年根據市場水位重算支出

提領不是固定金額 + CPI 調整，而是**每年根據當年組合市值 × 估值指標對應 SWR 重新計算**。市場漲了自動花更多，市場跌了自動縮減，從根本切斷「熊市持續失血」的惡性循環。

### 1. 每年支出計算公式

$$\text{當年支出} = \text{當年組合市值} \times \text{估值指標對應 SWR}$$

### 2. CAPE 對應提領率（SPY / VT 適用）

| 市場估值 (Shiller CAPE) | 適用 SWR | 戰略邏輯 |
| :--- | :--- | :--- |
| **> 30（昂貴）** | **4.7%** | 防守模式，應對潛在的報酬順序風險 |
| **15 – 30（合理）** | **5.2%** | 標準模式，資產增長機率較高 |
| **≤ 15（便宜）** | **6.0%** | 進攻模式，此時提領對資產長線損害最小 |

### 3. 台灣 P/E 對應提領率（0050 專屬選項）

| 市場估值 (台灣本益比 P/E) | 適用 SWR | 戰略邏輯 |
| :--- | :--- | :--- |
| **> 20（昂貴）** | **4.7%** | 防守模式 |
| **13 – 20（正常）** | **5.2%** | 標準模式 |
| **≤ 13（危機底部）** | **6.0%** | 進攻模式 |

> **與傳統 Bengen 的差異：** 傳統做法是「退休第一年定好金額，之後逐年乘以 CPI」，本策略改為每年動態重算，支出與資產規模連動，避免固定高提領在熊市不斷侵蝕本金。

## 三、回測資產選項

| 資產 | 說明 | 歷史資料來源 |
| :--- | :--- | :--- |
| **SPY** | S&P 500 | 全歷史（1966–2023）真實數據 |

| **SSO+SGOV** | 50% SSO（2× 槓桿）+ 50% SGOV（短債） | 2006 前採 2×S&P500 代理；SGOV 採美國3月短債利率代理 |

> ⚠️ **重要限制：** SWR 計算永遠使用**美股席勒 CAPE** 作為基準（0050 選 P/E 模式除外）。VT 與 0050 的模擬結果應視為參考，而非精確預測。

## 四、SSO+SGOV 策略：波動收割 + 再平衡紀律

### 策略邏輯

**50% SSO + 50% SGOV** 是一種「雙槓桿+無風險資產」的極端波動收割策略。核心並非持有 SSO 等待上漲，而是透過**強制再平衡**把波動轉化為獲利：

1. SSO 劇烈上漲 → 賣 SSO、買 SGOV，鎖定槓桿利潤
2. SSO 劇烈下跌 → 賣 SGOV、買 SSO，低點加碼槓桿

### 再平衡觸發條件

> **|SSO 年報酬| > 20%** 時執行再平衡，使兩者回到 50/50

- 再平衡次數估算公式：`floor(|SSO 年報酬| / 20%)`
  - SSO +47% → 約 2 次再平衡
  - SSO -65% → 約 3 次再平衡
  - SSO +10% → 0 次，不觸發

### 歷史特性

| 特性 | 數值 |
| :--- | :--- |
| 1999 年起始（30 年模擬）| 再平衡觸發約 24 年（80% 的年份） |
| 最劇烈年度 | 2008（SSO -65.7%，3 次再平衡） |
| 最強牛市 | 2013（SSO +68.4%，3 次再平衡） |
| 與 SGOV 搭配優勢 | 高利率環境（2022-2023）SGOV 年化 5%+，拉高組合底部 |

### 風險提示

- **不適合台灣居民直接持有**（SSO 為美國 ETF，台灣券商無法質押）
- 2× 槓桿在連續下跌中有「槓桿衰減」問題，長期持有不等於 2× 長期報酬
- 此策略為**純回測模型參考**，對應的提領護欄規則與其他資產相同

## 五、每年執行順序（完整版）

以下為程式模擬每一年的計算步驟：

```
Step 1：取得當年報酬率（ret）與通膨率（CPI）
        ├─ SSO+SGOV：查 SSO_SGOV_DATA 表
        └─ 其他資產：查對應歷史數據

Step 2：通膨調整提領金額
        ├─ 若 ret ≥ CPI → 提領金額不變
        └─ 若 ret < CPI → 提領金額 × (1 + CPI%)
           （補償購買力，不讓生活費實質縮水）

Step 3：判斷質押資格（任一否決 → 停止質押，轉賣股）
        ├─ CPI > 4%：高通膨，禁止新增借款
        ├─ 維持率 < 300%：資產/負債比過低
        ├─ 借款 ≥ 初始資產 × N%：達債務天花板
        └─ 估值條件不符（CAPE/P/E 超出設定範圍）

Step 4：執行提領
        ├─ 可質押 → 借款支付生活費（累積借款增加）
        └─ 不可質押 → 股息先抵利息，差額賣股補足

Step 5：應用年度報酬 → 更新資產市值

Step 6：以「下一年」估值重算 SWR → 決定明年提領金額
```

## 六、四重護欄（任一觸發切換賣股）

| 優先 | 護欄名稱 | 觸發條件 | 程式代號 |
| :--- | :--- | :--- | :--- |
| ① | 通膨護欄 | CPI > 4% | `"cpi"` |
| ② | 估值條件 | CAPE / P/E 超出使用者設定範圍 | `"valuation"` |
| ③ | 維持率護欄 | 市值 / 借款 < 300% | `"maintenance"` |
| ④ | 債務天花板 | 借款 ≥ 初始資產 × N% | `"debt-cap"` |

> 所有護欄解除後（CPI 降回 ≤ 4%、維持率回升、借款縮減），系統**自動恢復質押模式**，不需手動重設。

### 護欄觸發後的賣股公式

**淨股息 = 當年股息收入 − 累積借款利息**

| 情境 | 賣股金額 |
| :--- | :--- |
| 股息 > 利息（淨股息 > 0） | `max(0, 生活費 − 淨股息)` |
| 股息 < 利息（淨股息 < 0） | `生活費 + |淨股息|` |

> 先讓股息抵扣利息，剩餘缺口才賣股。護欄期間累積借款金額**不再增加**，舊債靠股息持續攤銷利息。

## 七、通膨調整機制詳解

### 提領金額調整範例（SWR = 4.7%）

| 當年報酬 | CPI | 觸發調整？ | 調整後等效 SWR |
| :--- | :--- | :--- | :--- |
| +5% | 2% | 否（5% ≥ 2%） | 4.7%（不變） |
| +1% | 3% | 是（1% < 3%） | 4.7% × 1.03 = **4.84%** |
| -18% | 8% | 是（-18% < 8%） | 4.7% × 1.08 = **5.08%** |

> 2022 年 SPY 案例（-18.11%，CPI 8%）：提領率從 4.7% 上浮至 **5.08%**，補償高通膨時期的購買力損失。

### 設計邏輯

調整幅度刻意保守。目標不是「跑贏通膨」，而是「不讓購買力因一時的資產低迷而永久縮水」。市場若隔年回升，動態 SWR 自然恢復，不需要額外機制重設。

## 八、CAPE 條件式質押（進階設定）

程式提供三段切換，允許使用者自訂「何種估值環境才允許質押」：

| 設定 | 效果 |
| :--- | :--- |
| **不限** | 無論 CAPE 高低均可質押（傳統模式） |
| **CAPE <** + 數值 | 僅市場便宜時才質押（推薦：CAPE < 15） |
| **CAPE >** + 數值 | 僅市場昂貴時才質押（一般不建議） |

0050 P/E 模式下，條件按鈕自動切換為 **P/E <** / **P/E >**。

**推薦設定：CAPE < 15**，對應「股票最珍貴、每股未來報酬期望值最高」的歷史低估水位（約 26 年一遇）。

## 九、壓力測試：1999 年退休情境

最壞的歷史起點（CAPE 41.3）：2000–2002 網路泡沫（-43%）+ 2008 金融海嘯（-37%），連遇兩次重擊。

- **60 歲（2009 年）是壓力頂點：** 維持率跌破 300% 或觸及債務天花板，護欄自動啟動
- **此時 CAPE 僅 15.1**，市場處於歷史低估區，賣股雖痛，每年賣出的僅是「生活費 − 淨股息」的缺口，非整批清倉
- **後續強力反彈：** 動態 SWR 在低點自動壓縮支出，保護存活本金，2010 年後市場複利修復資產

**關鍵應對原則：** 不做系統外的決策（不恐慌全清）、考慮暫時降支出至 4.7% 水準、等待維持率自然回升後恢復質押。

## 十、2026 年實務備忘錄

> **執行時機：** 本策略預計於 2029 年 Grand Park 落成後，辛亥路與哈爾濱街陸續出脫、流動金融資產比重大幅提升後，才進入全速執行階段。2029 年前以模擬回測為主，逐步建立可質押部位。

1. **契約管理：** 注意台灣券商質押通常為「1.5 年 + 1.5 年」滾動。須確保有「備援券商」應對合約到期時的轉質需求。
2. **標的監控：** 質押標的必須是大盤型 ETF。若標的被剔除於券商質押清單，需有備用的現金緩衝空間。
3. **債務天花板設定：** 預設 25%，可依個人風險承受度在回測程式中調整。建議從壓力最大的歷史起點（1999 年）反覆測試，找出不觸發斷頭的最大值。
4. **心態修正：** 記住，4.7% 是保險，質押是工具。真實的財務自由不在於資產數字，而在於「面對崩盤時依然能穩定產生現金流」的掌控感。

## 十一、批判性觀點與主要風險

| 風險因素 | 影響程度 | 應對措施 |
| :--- | :--- | :--- |
| 質押利率 > 股息殖利率 | 高 | 回歸賣股提領，停止質押 |
| 市場連續下跌 > 30% | 高 | 維持率護欄 + 債務天花板雙重保護 |
| 退休時 CAPE 極高（> 40） | 高 | 天花板設保守值（如 15–20%），降低最大槓桿 |
| SSO+SGOV 槓桿衰減 | 高 | 僅作回測參考，非台灣實盤推薦策略 |
| ETF 被剔除質押清單 | 中 | 備援券商 + 現金緩衝 |
| 合約到期無法續約 | 中 | 提前 6 個月安排轉質 |
| 惡性通膨 CPI > 4% | 中 | 四重護欄第①條自動攔截，停止新增借款 |

> **最大系統性風險：** 若全球進入超高利率時代（質押利率 > 5%），「股息覆蓋利息」的核心假設失效，屆時需果斷回歸傳統 Bengen 提領法，停止所有質押。

## 十二、回測程式功能說明（2026-04-10 版）

### 主要參數

| 參數 | 說明 |
| :--- | :--- |
| 初始資產 | 退休時的流動金融資產總額 |
| 退休年齡 / 預期壽命 | 決定模擬總年數 |
| 股息率 | ETF 年化股息殖利率（0050 約 3–4%，SPY 約 1.5%） |
| 質押利率 | 券商質押借款年利率 |
| 安全維持率門檻 | 低於此值停止質押（預設 300%） |
| 累積借款上限 | 初始資產的 N%（預設 25%） |
| 估值指標 | CAPE（SPY/VT）或台灣 P/E（0050 可選） |
| 質押 CAPE/P/E 條件 | 設定只在特定估值區間才質押 |
| 歷史起點 | 1966–2023 任意年份 |

### 統計卡說明

| 統計卡 | 說明 |
| :--- | :--- |
| 終局淨資產 | 模擬結束時的 市值 − 負債 |
| 終局資產市值 | 模擬結束時的股票組合市值 |
| 終局累積債務 | 模擬結束時尚未償還的質押借款 |
| 質押年度 | 當年策略為「借款支付生活費」的年份列表 |
| 賣股年度 | 當年護欄觸發、改為賣股的年份列表 |
| SSO 再平衡年度 | 僅 SSO+SGOV 資產：顯示觸發再平衡的年份及次數（如 `2008(3次)`） |
| 通膨攔截年度 | CPI > 4% 觸發停止質押的年份列表 |

### Tooltip 護欄原因對照

| 顯示文字 | 觸發原因 |
| :--- | :--- |
| `通膨CPI N%` | 當年 CPI 超過 4% |
| `估值條件` | CAPE 或 P/E 超出使用者設定範圍 |
| `維持率` | 市值/借款比率跌破門檻 |
| `借款上限` | 累積借款達初始資產 × 上限比例 |
